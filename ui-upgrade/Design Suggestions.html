Key observation: Style B ("Clean") is closest to the right answer — it's readable, standard density, and doesn't fight the content. The fix is to migrate Styles A and C toward Style B, not to invent a fourth style. The "Aura" premium aesthetic is appropriate only for the sidebar and dashboard header, not for data-entry forms and tables.

02 Typography hierarchy collapse Critical
Style A uses font-black uppercase tracking-[0.2em] for every string — labels, values, buttons, sub-text
When everything is the same visual weight, users can't scan. This problem is concentrated in Style A modules (Dashboard, Expenses). Style B and C are actually better here — their label weight is more restrained. The fix is to align all modules to the lighter convention.

03 Stat card design Critical
rounded-3xl + gradient icon + hover:-translate-y-1.5 shadow-2xl = consumer SaaS, not school ERP
These cards feel like a marketing landing page. Staff open this dashboard 50+ times daily. Enterprise data cards should be calm — let the numbers be the hero, not the card chrome. The hover lift animation (hover:-translate-y-1.5) on clickable data is particularly out of place.

04 Sidebar active state Important
Solid bg-indigo-600 pill vs faint sub-nav tint — two different visual languages
Top-level active nav items use a full indigo background pill. Sub-nav active items use a text-color-only treatment. These look like different components. A left-border indicator unifies both levels and reduces the sidebar's visual weight.

05 Dashboard header gradient Important

The page has three layers of indigo: the sidebar, the header banner, and the stat card icons. There's no visual rest. The header's only job is orientation (school name, date, quick actions) — it should be the quietest element on the page, not the loudest.

06 Floating glassmorphism topbar Important
mx-6 mt-3 rounded-xl — a floating pill topbar that wastes ~10px of vertical space on every page
The topbar floats disconnected from both the sidebar and the content below. On smaller screens the gap shrinks awkwardly. Backdrop-blur also causes visual glitches on some browsers. A standard flush border-b bar is cleaner, more professional, and more space-efficient.
/* DashboardLayout.tsx — header element */ "aura-glass sticky top-0 h-14 border-b border-slate-200/50 flex items-center justify-between px-3 sm:px-6 z-40 shrink-0 print:hidden mx-2 sm:mx-6 mt-2 sm:mt-3 rounded-xl shadow-lg shadow-slate-200/20" "bg-white sticky top-0 h-14 border-b border-gray-200 flex items-center justify-between px-6 z-40 shrink-0 print:hidden"
Also remove the aura-glass class from index.css from this element — backdrop-filter is not needed on a white bar.
07 Color fragmentation Polish
3 separate color systems that don't reference each other — plus Style C introduces amber as a brand color
COLORS[] in Dashboard for charts, colorMap with 11 Tailwind class names, inline hex values in chart gradients, and amber as a module accent in Result pages — none of these talk to the CSS custom properties defined in index.css. Consolidate into one file.
// NEW: src/constants/colors.ts — one source of truth export const SEMANTIC = { primary: { hex: '#6366f1', bg: 'bg-indigo-50', text: 'text-indigo-600' }, success: { hex: '#10b981', bg: 'bg-emerald-50', text: 'text-emerald-600' }, warning: { hex: '#f59e0b', bg: 'bg-amber-50', text: 'text-amber-600' }, danger: { hex: '#ef4444', bg: 'bg-rose-50', text: 'text-rose-600' }, neutral: { hex: '#6b7280', bg: 'bg-slate-50', text: 'text-slate-600' }, } as const; // Chart colors — array format for Recharts export const CHART_COLORS = Object.values(SEMANTIC).map(s => s.hex); // Dashboard.tsx — replace hardcoded COLORS array const COLORS = ['#10b981', '#f59e0b', '#ef4444', '#6366f1']; import { CHART_COLORS } from '../constants/colors'; // Removes the need for the 11-entry colorMap as well
Note on amber: The amber accent in Result pages (Style C) came from Codex treating amber as the "exam" color. Don't carry this forward — it conflicts with amber being used for warnings throughout the rest of the app. Result module table headers should use the standard bg-gray-50.
Phased remediation plan
Strategy: Don't try to fix everything at once — it will introduce regressions across 60+ files. Work module-by-module, using Style B ("Clean") as the target. The sidebar and global chrome (issues 04–06) give the biggest cross-app impact for the least effort — do those first.

These changes touch the shell, not individual pages, so they improve every module instantly with minimal risk of regression.
